# Switch-On 다이어트 웹 서비스

## 프로젝트 개요
다이어트를 함께 하는 사람들이 모여 챌린지를 수행하고, 커뮤니티를 통해 인증 및 정보를 공유하는 웹 서비스

### MVP 범위
- 스위치온 다이어트 (간헐적 단식 + 식단 인증) 전용

---

## 기술 스택
- **백엔드:** Spring Boot 3.x + Java 17
- **프론트엔드:** Next.js 14 (App Router)
- **DB:** PostgreSQL (운영) / H2 (개발)
- **인증:** JWT + 이메일/비밀번호
- **파일 저장:** S3 (MVP는 로컬 저장)

---

## 개발 계획

### Phase 1: 프로젝트 초기 설정
- [ ] Spring Boot 프로젝트 생성 (`backend/`)
- [ ] Next.js 프로젝트 생성 (`frontend/`)
- [ ] 공통 설정 (CORS, Security, DB 연결)

### Phase 2: 사용자 관리
- [ ] 회원가입/로그인 API
- [ ] JWT 토큰 발급/검증
- [ ] 프로필 관리

### Phase 3: 챌린지 시스템
- [ ] 챌린지 생성 (스위치온 다이어트)
  - 기간 설정
  - 참가 인원 제한
  - 디파짓 금액 설정
- [ ] 챌린지 참여 신청
- [ ] 디파짓 결제 (Mock API)

### Phase 4: 미션 수행
- [ ] 간헐적 단식 체크
  - 단식 시작/종료 시간 기록
  - 일일 단식 달성 여부 체크
- [ ] 식단 인증
  - 사진 업로드
  - 시간대별 인증 (아침/점심/저녁)

### Phase 5: 커뮤니티
- [ ] 인증 피드 (타임라인)
- [ ] 댓글/좋아요
- [ ] 정보 공유 게시판

### Phase 6: 정산 시스템
- [ ] 미션 달성률 계산
- [ ] 디파짓 환급/몰수 로직 (Mock)

---

## 데이터베이스 설계

### 주요 테이블
- `users` - 사용자
- `challenges` - 챌린지
- `challenge_participants` - 챌린지 참가자 + 디파짓
- `daily_missions` - 일일 미션 기록
- `meal_certifications` - 식단 인증
- `posts` - 커뮤니티 게시글
- `comments` - 댓글
